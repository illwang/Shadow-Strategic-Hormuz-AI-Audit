## Chapter 1 Information Architecture Layer

### 1.1 Three-Layer Physical Isolation Structure

| Layer | Definition | Admission Threshold | Usage Boundary |
|:---|:---|:---|:---|
| **T1 Confirmation Layer** | Facts that have occurred and can be independently reproduced | Must have ≥1 🟢 or 🟡 free verification path; multi-source public verification, confidence ≥85% | Paid database exclusive verification prohibited |
| **T2 Inference Layer** | Reasonable deductions based on T1 | Directional judgment (60-85%) or speculative observation (≤60%); paid research summaries allowed as auxiliary | Must label source and limitations |
| **T3 Narrative Layer** | Unverifiable rumors, market sentiment, political trial balloons | Prohibited from entering main text | Only in appendix "Unverified Signal Observation Zone", label source and upgrade conditions |

### 1.2 OSINT Verifiability Principle

- T1 confirmatory judgments must attach at least one free verification path (URL / public database / API / official announcement)
- Data points verifiable only through 🔴 paid databases shall not enter T1, only as T2 or T3
- **Analysts do not hold, read, or cite any classified/paid report originals**
- Accept "verification conclusions" from external institutions (e.g., "deviation ~15%"), but not classified data originals
- All externally published content remains **100% OSINT-based**

### 1.3 Source-Content Cross-Rating Matrix (Rule Zero)

```
                Content Credibility
                   A         B         C
            ┌─────────┬─────────┬─────────┐
         S  │   S1    │   S2    │   S3    │  ← Official/Primary data
Source      ├─────────┼─────────┼─────────┤
Level       N  │   N1    │   N2    │   N3    │  ← Mainstream wire services/think tanks
            ├─────────┼─────────┼─────────┤
         C  │   C1    │   C2    │   C3    │  ← Self-media/Rumors
            └─────────┴─────────┴─────────┘
```

> **Admission Threshold**: Signals below C3 are prohibited from AIPI calculation; must reach B2 or above.

### 1.4 Argumentation Strength Grading System

| Grade | Confidence | Usage Standard |
|:---|:---|:---|
| Confirmatory Judgment | ≥85% | Multi-source public verification, can enter T1 |
| Directional Judgment | 60%-85% | Reasonable deduction based on T1, can enter main text |
| Speculative Observation | ≤60% | Logical deduction, must explicitly label uncertainty |
| Analytical Inference | Logical deduction | Scenario-based expression, **probability percentages prohibited** |

**Three-Nos Principle**:
1. Do not assign probabilities to qualitative events
2. Do not apply mechanical formulas to mechanisms
3. Do not conceal "unquantifiable" areas

### 1.5 Reproducibility Boundary Declaration

This framework pursues **process traceability and divergence locatability**, not mechanical identity of results. The reproducibility of analysis steps is divided into three levels:

| Level | Content | Reproducibility | Description |
|:---|:---|:---|:---|
| Fact Layer (T1) | Factual basis of all confirmatory judgments | ✅ Strongly Reproducible | All require cross-verification by two+ independent sources; anyone can independently reproduce |
| Calculation Layer | AIPI weighted aggregation, elasticity weighted calculation, scenario quantitative deduction | ✅ Strongly Reproducible | Weights and formulas are public; given score inputs can be recalculated |
| Mapping Layer | Fact→sub-dimension score, arrow→numerical mapping | ⚠️ Weakly Reproducible | Anchor tables provide interval constraints; different analysts' scores have reasonable fluctuation (typically within ±0.10), but direction and magnitude are consistent |
| Judgment Layer | Weight selection, scenario hypothesis construction, strategy priority ranking | ❌ Not Reproducible | Belongs to analyst's strategic judgment; affected by perspective and stance |

**Framework's Strategy for "Non-Reproducibility"**:
- Do not pursue eliminating non-reproducibility (that would lead to false precision), but **expose and make it explicit**
- Through change audit tables, convergence verification records, falsification conditions, etc., make non-reproducible judgments subject to **factual testing** ex post
- Divergence among different analysts is treated as **signal** rather than **noise** — if multiple analysts' score differences for the same event exceed ±0.15, it indicates fundamental interpretive uncertainty exists and should trigger deep analysis

**User Responsibility Boundary**: AIPI levels are analytical auxiliary signals and should not be treated as mechanical decision directives. Final responsibility for any decisions based on this report rests with the decision-maker.

### 1.6 Six Permanent Validation Rules

**Rule One: Semiconductor New Architecture Mass Production Cycle Validation**  
New chip architecture mass production automatically applies an **18-month buffer period**. During the buffer period, any "imminent mass production" statement must be downgraded to T2 or T3, prohibited from entering T1.

**Rule Two: Material Substitution Elasticity Three Questions**  
Any material control statement must answer:
1. Does functional substitution exist (chemical/electrical vs. mechanical/structural)?
2. How long is the certification cycle?
3. How long can inventory buffer last?
> If any answer is "no/unknown/insufficient", that material bottleneck enters T1.

**Rule Three: Geopolitical Statement Physical/Legal Feasibility Dual Filter**
- **Physical Layer**: Do technology, logistics, and infrastructure support execution?
- **Legal Layer**: Do international law, domestic law, and contractual frameworks grant legitimacy?
> If either fails, downgrade to posture (T3), do not enter T1/T2.

**Rule Four: Financial Revenue Quality Triple Breakdown**
Before any financial data enters T1, it must be broken down:
1. Cash/Credit — Is accounts receivable ratio abnormal?
2. Related Party/Third Party — Is revenue from related-party transactions?
3. Consumed/Committed — Is revenue delivered or merely contract-signed?

**Rule Five: Lagging Data Declaration**
When involving official lagging data, if using third-party estimates, must label:  
> "Pending official data verification, expected release date [date]"

**Rule Six: Signal Pollution Isolation Zone**
The following five signal types are prohibited from directly entering core judgments; must be cleaned before proceeding through Rules Zero to Five:
1. Market sentiment noise
2. Political trial balloons
3. Corporate PR statements
4. Social media amplification effects
5. Capital narrative manipulation

## Chapter 2 Strategic Pressure Assessment Layer

### 2.1 Seven-Layer Strategic Pressure Map (L0-L6)

**Design Philosophy**: The true constraints of the AI industrial chain come from the intersection of immovable resource layers (geography, energy, environment) and replaceable device layers (chips, materials). Layering follows "physical constraint progression relationships", not simple supply chain upstream-downstream.

| Layer | Code | Constraint Nature | Coverage |
|:---|:---|:---|:---|
| **L0** | GEO | Immovable | Advanced process geographic concentration, geopolitical conflict, capacity dispersion, drought/earthquake physical vulnerability |
| **L1** | CHIP | Core Devices | HBM capacity, advanced packaging (CoWoS/Chiplet), domestic substitution progress, logic chips |
| **L2** | CONN | Interconnect Bottleneck | EML/optical modules, silicon photonics, CPO, network architecture transformation |
| **L3** | POWER | Energy & Physical Security | Diesel generators/transformers/power grid, data center physical security, SMR/micro-reactors, BESS |
| **L4** | MAT | Materials | Helium, photoresist, hydrogen fluoride, silicon wafers, rare gases, rare earths |
| **L4.5** | PASS | Passive Components (Full-chain inflation spread signal layer) | MLCC, resistors, inductors |
| **L5** | ENV | Environmental Compliance | Carbon emissions, liquid cooling penetration, water resources, AI Act, environmental assessment approval |
| **L6** | FIN | Financial Layer | CAPEX sustainability, war risk premiums, financing environment, valuation bubbles, operating cash flow |

### 2.2 Inter-Layer Coupling Status Analysis Standard

**Inter-Layer Coupling Rule**: L4 material supply interruption → L1 packaging shutdown → L0 geopolitical escalation. Any single-layer analysis must label its cascading risk to adjacent layers.

Each active coupling line must answer three questions:
1. **What is the trigger condition?** (Quantitative threshold or specific event)
2. **What is the transmission mechanism?** (Physical/financial/policy/information)
3. **Is the current state real pressure change, rater calibration adjustment, or methodology correction?**

**Coupling Status Classification**:
- `Dormant` — No significant coupling currently
- `Active` — Direct transmission path exists
- `Internally Active` — Sub-modules within same layer are linked
- `Alert` — Cross-layer risk may transmit systemically

**Five-Step Derivation Method (applicable to hidden layers like L4/L4.5)**:
1. Discover signals (price hikes, controls, inventory anomalies)
2. Cross-verify (exclude single explanations, verify global capacity distribution)
3. Assess strategic relevance (specific application scenarios of this material in AI industrial chain)
4. Judge "combination punch" risk (whether it forms joint restrictions with other categories)
5. Incorporate into monitoring framework (set trigger thresholds and upgrade paths)

### 2.3 AIPI Strategic Pressure Level (AIPI-Level)

**Formula**:

```
AIPI = S×0.40 + D×0.35 + G×0.25
```

**Collinearity Control**: When S and G change simultaneously due to the same event, take the larger change, and the other dimension is counted at **50%**. *(Note: The 50% discount coefficient is an **analytical inference**; calibration in actual application is recommended.)*

**Limitations**: Subjectivity, ±0.08 error, OSINT boundary, extreme event failure. Cross-verification with paid reports is recommended.

**S (Supply Rigidity) Sub-dimension**:

| Sub-dimension | Weight | Description |
|:---|:---|:---|
| Core Immediate Supply (HBM/Advanced Process) | 0.35 | Currently most constrained link |
| Advanced Packaging Capacity | 0.20 | CoWoS/Chiplet physical ceiling |
| Optical Interconnect/High-Speed Transmission Devices | 0.15 | EML/silicon photonics bottleneck |
| Energy Infrastructure | 0.10 | Transformers/diesel generators/power grid connection |
| Semiconductor Materials | 0.15 | Helium/photoresist/hydrogen fluoride |
| Passive Components | 0.05 | MLCC and other inflation spread signals |

**D (Demand Verification) Sub-dimension**:

| Sub-dimension | Weight | Description |
|:---|:---|:---|
| Closed-Source Model Commercialization Quality | 0.35 | ARR/FCF/revenue structure |
| Open-Source Model Efficiency Improvement | 0.25 | MoE/heterogeneous computing/HBM bypass |
| Cloud Provider/Head Customer CAPEX Sustainability | 0.25 | CAPEX growth rate vs. cloud revenue growth rate |
| Emerging Demand Poles | 0.15 | Embodied intelligence, edge computing, etc. |

**G (Geopolitical Fracture) Sub-dimension**:

| Sub-dimension | Weight | Description |
|:---|:---|:---|
| US-China Tech Game | 0.30 | Controls/approvals/technology blockade |
| Material Export Control Confrontation | 0.25 | Photoresist/hydrogen fluoride/rare earth mutual countermeasures |
| Key Production Region Labor/Geopoliticization | 0.20 | Strikes, earthquakes, droughts |
| Physical Vulnerability | 0.15 | Taiwan Strait/TSMC concentration risk |
| Shipping/Energy Geopolitical Fracture | 0.10 | Strait blockade, war risk premiums |

**Scoring Standard**: Each sub-dimension 0-1 points, 1=fully safe/loose, 0=fully broken/extreme tension

**Composite Score Range**:

| Level | Range | Status Definition |
|:---|:---|:---|
| AIPI-A | 0.80-1.00 | Strategic loose state |
| AIPI-B | 0.70-0.79 | Moderate pressure |
| AIPI-C | 0.60-0.69 | Structural tension |
| AIPI-D | 0.50-0.59 | Strategic fragile state |
| AIPI-E | <0.50 | Extreme destabilization |

### 2.4 Score Anchor Standard (0-1 Scale)

| Score Range | Status Label | S Typical Scenario | D Typical Scenario | G Typical Scenario |
|:---|:---|:---|:---|:---|
| 0.80-1.00 | Extreme Pressure | Core supplier shutdown >72hrs; lead time extended >2x with no alternative | Head cloud providers cut CAPEX >20% for two consecutive quarters; ARR growth turns negative | Comprehensive embargo effective; key strait military blockade |
| 0.60-0.79 | Significant Pressure | Lead time extended 50%-100%; inventory <4 weeks; price hike letters spread | CAPEX growth slows to single digits; ARR and token cost scissors gap widens | Partial embargo/control escalation; key production region strikes or political instability |
| 0.40-0.59 | Moderate Pressure | Lead time extended 20%-50%; individual categories short; alternatives in certification | CAPEX growth slows but still growing; some customers delay procurement | Trade friction continues but not escalated; individual countries tighten export review |
| 0.20-0.39 | Mild Pressure | Market tight balance; prices rise moderately; alternatives certified | Some downstream customers wait-and-see; open-source models reduce some inference demand | Trade friction eases or maintains status quo |
| 0.00-0.19 | Normal Level | Supply abundant; lead times normal; inventory sufficient | AI demand continues growing; ARR growth stable; CAPEX returns acceptable | Free trade normal; no major geopolitical disturbance |

**Usage Rules**:
- Sub-dimension scores are analyst subjective judgments based on public facts, anchored to above typical scenarios
- Scores taken to two decimal places (multiples of 0.05)
- Each scoring must note referenced factual basis in "Basis" column
- When scoring deviation exceeds ±0.10, attribution must be explained in change audit table

### 2.5 Scenario Arrow → Numerical Mapping Rules

| Arrow Symbol | Score Change Range | Typical Trigger Condition |
|:---|:---|:---|
| ↑↑↑ | +0.15 to +0.25 | Core supply chain interruption (strike restart, comprehensive embargo, major natural disaster causing shutdown >48hrs) |
| ↑↑ | +0.10 to +0.15 | Supply significantly tightens (lead time doubles, new control measures effective, inventory drops below 2 weeks) |
| ↑ | +0.05 to +0.10 | Pressure mildly rises (lead time extends 20%-50%, individual suppliers raise prices, rumor-level control discussions) |

|:---|:---|

### 2.7 Sensitivity Deduction & Concurrent Scenarios

Must include four types of deduction:
1. **Single-variable deduction**: Upside and downside scenarios for each key event
2. **Dual-variable concurrent deduction**: Net effect of two key events occurring simultaneously
3. **Extreme concurrent deduction**: Systemic effects of three+ variables impacting simultaneously
4. **Efficiency Revolution/Technology Disruption Independent Monitoring**: Establish trigger thresholds and action framework

> **Limitation Declaration (Mandatory)**: Subjective judgment, ±0.08 underlying uncertainty, OSINT boundary, extreme event failure. When any L0 or L1 core sub-dimension reaches "extreme destabilization" condition, directly activate S-level supplementary report.

## Chapter 4 Response Strategy Layer

### 4.1 Geo-Economic Elasticity Model

**Elasticity Level Quantification Standard (Decile Weighted)**:

```
Composite Score = 0.50 × Supply Stability + 0.25 × Substitution Speed + 0.25 × Cost Affordability
```

| Level | Range |
|:---|:---|
| ★★★★★ | ≥9.0 |
| ★★★★ | 7.5-8.9 |
| ★★★ | 6.0-7.4 |
| ★★ | 4.5-5.9 |
| ★ | <4.5 |

**Elasticity Level Non-linear Penalty Rules**:
- Supply stability <5 AND substitution speed <5 → Elasticity level cannot exceed ★★
- Substitution speed <3 → Elasticity level ceiling ★★★

### 4.3 Regret Value Assessment Standard

For each elasticity strategy, assess regret level under **four endgame scenarios**:

1. **Soft Landing**: AI ARR quality reliable, continuously exceeds cost, FCF turns positive

3. **Capital Dry-up**: CAPEX negative growth for two consecutive quarters, operating cash flow cannot cover
4. **Geopolitical Fracture**: Core supply fully embargoed

> Label current regret value using worst scenario score, helping decision-makers examine strategy portfolio robustness from a "regret minimization" perspective.

### 5.6 Convergence Verification Process

**Purpose**: Systematically compare each period's predictions with actual results, identify framework calibration deviations, prevent "always correct" narrative self-consistency.

**Operation Process (execute when each report is published)**:

1. **Extract last period key predictions**: From last period AIPI change audit table and sensitivity deduction, extract at least 3 verifiable prediction directions (no ambiguous expressions allowed)
2. **Compare with current period actual**: Compare prediction directions with actual occurred directions one by one
3. **Quantify deviation**: If prediction involves AIPI numerical change, calculate difference between predicted change and actual change
4. **Classify attribution**: Deviation归入 one of three categories:
   - **Unexpected event**: Black swan event unforeseeable at prediction time occurred
   - **Weight misalignment**: Event prediction direction correct, but impact on AIPI overestimated or underestimated
   - **Methodology defect**: Framework itself failed to capture key transmission path
5. **Trigger calibration**: If two consecutive periods show same-direction deviation (persistent overestimation or persistent underestimation), trigger weight or anchor table re-evaluation
